## Strategy

1. 접근 방식 
   - 2N개의 스티커 중 점수의 합이 최대가 되도록 스티커 떼기 (변을 공유하는 스티커는 동시 선택 불가)
   - 특정 열(`c`)에서 스티커를 선택하는 경우의 수 : 3 -> `위쪽 선택`, `아래쪽 선택`, `선택 안 함`
   - `현재 열까지의 스티커를 고려했을 때 점수 합 최대 = Max( 이전 열 다른 행 + 현재 위치 스티커 점수, 이전 열 같은 행 (현재 위치 스티커 포함 x))`

2. Top Down 재귀 방식 -> 상향식으로 변경 
   - $N$이 최대 100,000
   - 재귀를 활용한 Top-Down 방식은 자바 콜 스택 한계로 인해 런타임 에러가 발생
   - 바텀업으로 변경 

<br>

## Solution

```java
import java.io.*;
import java.util.*;

public class BOJ9465 {

    public static void main(String[] args) throws IOException{

        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        StringBuilder sb = new StringBuilder();
        StringTokenizer st;

        int T = Integer.parseInt(br.readLine());
        
        for(int t = 0; t < T; t++){
            int N = Integer.parseInt(br.readLine());
            int[][] s = new int[2][N];
            int[][] dp = new int[2][N];

            for(int r = 0; r < 2; r++){
                st = new StringTokenizer(br.readLine());
                for(int c = 0; c < N; c++){
                    s[r][c] = Integer.parseInt(st.nextToken());
                }
            }

            dp[0][0] = s[0][0];
            dp[1][0] = s[1][0];

            for (int c = 1; c < N; c++) {
                // 윗줄 -> 이전 열 아랫줄 선택 + 현재 || 이번 열 건너뛰기
                dp[0][c] = Math.max(dp[1][c-1] + s[0][c], dp[0][c-1]);
                // 윗줄 -> 이전 열 윗줄 선택 + 현재 || 이번 열 건너뛰기
                dp[1][c] = Math.max(dp[0][c-1] + s[1][c], dp[1][c-1]);
            }

            int result = Math.max(dp[0][N-1], dp[1][N-1]);
            sb.append(result).append("\n");
        }
        System.out.println(sb);
    }
}
```
